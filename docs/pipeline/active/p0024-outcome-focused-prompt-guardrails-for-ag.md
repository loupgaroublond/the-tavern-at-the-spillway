## Stub
STUB — needs design investigation before implementation.

Agents and background processes need prompt-level guardrails that push work toward outcome-focused execution rather than mechanical step-following. When an agent receives a plan or assignment, the system should detect whether it includes a clear outcome and definition of done — or whether it's just a list of mechanical steps with no success criteria. When the outcome is missing or the agent lacks a commitment (in the Tavern sense), some mechanism should nudge the agent to request clarification before proceeding blindly. The exact intervention mechanism is TBD — could be prompt instructions, a pre-flight check in the agent protocol, commitment verification at assignment time, or something else entirely.
